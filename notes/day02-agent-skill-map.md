## 补充完善版

### 1. Agent 开发岗位需要哪些核心技能？

Agent 开发岗位的核心不是训练一个大模型，而是把 LLM 接入真实产品或业务流程，让它能稳定地理解任务、调用工具、检索知识、执行步骤，并且可以被评估、部署和持续优化。

我可以把 Agent 开发能力分成 8 层：

1. 基础编码能力

   需要掌握 Python / TypeScript、HTTP API、JSON、异步任务、错误处理、日志、测试和基本工程结构。Agent 最后一定要落到一个可运行的服务、脚本、应用或工作流里，所以编码能力是底座。

2. LLM API 调用能力

   需要理解 messages、system / user / assistant 角色、token、上下文窗口、temperature、top-p、stream、结构化输出等概念。只有理解这些，才能稳定地调用模型，而不是只会在网页里聊天。

3. Prompt 工程与任务拆解

   需要把模糊任务拆成清晰指令、步骤、输入格式和输出格式。Prompt 不是简单写一句“请帮我”，而是要告诉模型目标、上下文、约束、判断标准和失败时怎么处理。

4. Tool Calling 能力

   Agent 和普通聊天机器人的重要区别是可以调用外部工具。需要会设计工具的名称、说明、参数 schema、返回格式、错误处理和权限边界。例如搜索、数据库查询、文件读写、代码执行、日历、企业系统 API 都可以是工具。

5. RAG 与知识库能力

   LLM 本身不一定知道最新或私有信息，所以需要通过 RAG 把外部文档、业务资料、历史记录检索出来，再交给模型回答。这里需要理解 embedding、向量检索、chunk、召回、重排、引用来源和幻觉控制。

6. Workflow / Orchestration 能力

   真实任务通常不是一次模型调用就能完成，而是多个步骤组成的流程。需要知道什么时候用固定 workflow，什么时候用更自主的 agent loop，什么时候需要人工确认，什么时候要停止或回退。

7. Eval 与观测能力

   Agent 不能只靠主观感觉判断好坏。需要构造测试集，记录 trace，检查工具调用是否正确，评估回答质量、任务成功率、失败原因、成本和延迟。Eval 是持续优化 Agent 的核心能力。

8. Deploy 与工程化能力

   需要把 Agent 部署到真实环境里，处理环境变量、密钥、权限、安全、限流、监控、日志、成本控制和回滚。能本地跑起来只是第一步，能稳定在线上运行才算工程化。

### 2. Agent 开发岗和大模型算法岗有什么区别？

Agent 开发岗更偏应用工程，重点是“如何把模型用好”。它关心的是模型如何接入业务、如何调用工具、如何检索知识、如何完成多步骤任务、如何评估效果、如何部署上线。

大模型算法岗更偏模型本身，重点是“如何把模型训练好”。它关心的是模型结构、预训练、微调、对齐、强化学习、推理优化、训练数据和算力效率。

简单来说：

- 大模型算法岗：更接近“造发动机”。
- Agent 开发岗：更接近“用发动机造一辆能上路的车”。

我当前的学习目标更适合先走 Agent 应用开发路线。原因是我已经有一定编码基础，可以从调用 LLM API、Prompt、Tool、RAG、Workflow、Eval 这些工程能力开始积累，不需要一开始就进入模型训练和算法研究。

### 3. 我当前最需要补强哪些能力？

我目前的判断是：基础编码能力可以继续保持，但从 Prompt 之后的 Agent 核心工程能力都需要系统补强。为了让学习更可执行，我先按优先级拆成 3 个阶段。

第一优先级：Prompt 工程与任务拆解

这是最先要补的能力。因为无论后面做 Tool、RAG 还是 Workflow，都需要先把任务目标、上下文、步骤和输出标准说清楚。如果任务拆不清，后面的工具和流程也会混乱。

第二优先级：Tool Calling 与 Workflow

这是 Agent 区别于普通聊天机器人的关键。我要学会让模型不只是回答问题，而是能选择工具、调用工具、读取结果、继续下一步，并在失败时有处理策略。

第三优先级：Eval

Eval 是我之前容易忽略的部分，但它非常重要。没有 Eval，就只能凭感觉判断 Agent 有没有变好；有了 Eval，才能知道 prompt、工具、RAG 或流程改动是否真的提升了成功率。
